### A問題: 文字列操作

- 時間計算量: O(N)

### B問題: グリッド上の隣接セル数
**アプローチ**:
- H×Wのグリッドの各セルについて、マンハッタン距離が1のセル数を数える
- 4重ループで全ペアを調べる実装

**実装**:
```cpp
for (int y1 = 0; y1 < h; ++y1) {
  for (int x1 = 0; x1 < w; ++x1) {
    for (int y2 = 0; y2 < h; ++y2) {
      for (int x2 = 0; x2 < w; ++x2) {
        if (abs(x1 - x2) + abs(y1 - y2) == 1) {
          vec[y1][x1]++;
        }
      }
    }
  }
}
```

**時間計算量**: O(H²W²)

**改善点**:
- 4重ループは過剰な実装
- 各セルの隣接数は位置によって決まるパターンがある:
  - 角のセル: 2個
  - 辺のセル: 3個
  - 内部のセル: 4個
- O(HW)で済む問題をO(H²W²)で解いてしまった
- より効率的なアプローチを考える習慣をつけるべき

### C問題: 文字列中の'C'のカウント
**アプローチ**:
- 文字列中の各'C'について、両端からの距離の最小値+1を加算
- 各'C'が何通りの部分文字列に含まれるかを計算

**実装**:
```cpp
for (int i = 0; i < len; ++i) {
  char c = s[i];
  if (c == 'C') {
    ans += min(i, len - i - 1) + 1;
  }
}
```

**時間計算量**: O(N)

**学んだこと**:
- 各要素が貢献する回数を数える典型的な数え上げ問題
- 位置iの'C'は、左からi+1通り、右からlen-i通りの選び方があり、最小値を取る

### D問題: 優先度付きキューと中央値 (未AC)
**問題の理解**:
- 初期値xから始めて、q回のクエリを処理
- 各クエリで2つの数値を追加し、中央値を出力

**試したアプローチ**:
```cpp
priority_queue<int> half_before;  // 前半(大きい順)
priority_queue<int, vector<int>, greater<int>> half_after;  // 後半(小さい順)
```
- 2つのヒープで中央値を管理するという発想に至らなかった

**学ぶべきポイント**:
- 中央値を動的に管理する典型的なデータ構造(2つのヒープ)の正確な実装方法
- エッジケースの処理(偶数/奇数の要素数)
